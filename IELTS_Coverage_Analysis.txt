Type 1: MULTIPLE CHOICE
--------------------------------------------------------------------------------
IELTS Description:
- Question with four possible answers (A, B, C, D)
- OR first half of sentence with four possible endings
- Sometimes longer list with multiple answers to choose
- Questions in same order as text information

YOUR COVERAGE: ✅ FULLY COVERED

Block: MCQBlock.jsx
Capabilities:
- Single-answer multiple choice ✓
- Four options (A, B, C, D) ✓
- Review mode with correct/incorrect feedback ✓

GAPS:
- Multi-select variation (choose more than one answer) - NOT SUPPORTED
- Sentence ending format (could be handled by SentenceMatchingBlock)

RECOMMENDATION: Add multi-select mode to MCQBlock or create new MCQMultiSelectBlock

Type 2: IDENTIFYING INFORMATION (True/False/Not Given)
--------------------------------------------------------------------------------
IELTS Description:
- Given statements, must determine if they agree with text
- Answers: TRUE, FALSE, or NOT GIVEN
- FALSE = contradicts text
- NOT GIVEN = neither agrees nor contradicts

YOUR COVERAGE: ✅ FULLY COVERED

Block: TrinaryBlock.jsx (mode: 'tfng')
Capabilities:
- TRUE/FALSE/NOT GIVEN options ✓
- Review mode with feedback ✓
- Correct answer comparison ✓

NO GAPS IDENTIFIED

Type 4: MATCHING INFORMATION
--------------------------------------------------------------------------------
IELTS Description:
- Find specific information in paragraphs (A, B, C, etc.)
- Write letter of correct paragraph
- Not every paragraph may be used
- Some paragraphs may be used more than once
- Focuses on: specific details, examples, reasons, descriptions, comparisons, 
  summaries, explanations

YOUR COVERAGE: ⚠️ PARTIALLY COVERED

Block: MatchingChoiceBlock.jsx
Capabilities:
- Match questions to letter options ✓
- Review mode ✓

GAPS:
- No support for "use any letter more than once" mode
- No tracking of which paragraphs have been used
- No visual indication of reusable options
- No enforcement of single-use vs multi-use options

RECOMMENDATION: Enhance MatchingChoiceBlock with:
- allowReuse prop
- Visual indicator for "can be used more than once"
- Usage counter on options

--------------------------------------------------------------------------------
Type 5: MATCHING HEADINGS
--------------------------------------------------------------------------------
IELTS Description:
- List of headings (i, ii, iii, etc.)
- Match heading to correct paragraph (A, B, C, etc.)
- More headings than paragraphs (some not used)
- No heading used more than once
- May have example already matched

YOUR COVERAGE: ✅ FULLY COVERED

Block: HeadingMatchBlock.jsx
Capabilities:
- Match paragraphs to headings ✓
- Dropdown selection ✓
- Review mode ✓

GAPS:
- Uses Arabic numerals (1, 2, 3) instead of Roman numerals (i, ii, iii)
- No example pre-matching feature
- No enforcement of "no heading used more than once"

RECOMMENDATION: Minor enhancements:
- Add Roman numeral display option
- Add exampleMatch prop for pre-filled answers

Type 8: SENTENCE COMPLETION
--------------------------------------------------------------------------------
IELTS Description:
- Fill gaps in sentences using words FROM THE TEXT
- Word limit given (e.g., "NO MORE THAN TWO WORDS AND/OR A NUMBER")
- Answers must be exact words from text
- Questions in same order as text information
- Hyphenated words count as single words

YOUR COVERAGE: ⚠️ PARTIALLY COVERED

Block: GapFillBlock.jsx (partial)
Block: ShortAnswerBlock.jsx (partial)

GAPS:
- GapFillBlock uses word bank, not text extraction
- ShortAnswerBlock has no word limit enforcement
- No "words from text" validation
- No hyphenated word handling
- No number counting logic

RECOMMENDATION: Create new block or enhance ShortAnswerBlock:
- Add wordLimit prop with enforcement
- Add number counting (WORDS AND/OR A NUMBER)
- Add validation against text source
- Add hyphenated word handling

Type 9: SUMMARY/NOTE/TABLE/FLOW-CHART COMPLETION
--------------------------------------------------------------------------------
IELTS Description:
- Complete summary/notes/table/flow-chart using words from text
- TWO VARIATIONS:
  1. Select words directly from text (word limit applies)
  2. Choose from list of words (letters A, B, C)
- May be continuous text, notes, table, or flow-chart format
- Answers usually from one part of text

YOUR COVERAGE: ⚠️ PARTIALLY COVERED

Variation 1 (words from text):
- NO BLOCK EXISTS for this variation
- GapFillBlock uses word bank, not text extraction

Variation 2 (word list):
- GapFillBlock.jsx ✓ (covers this variation)

Format Support:
- Continuous text: GapFillBlock ✓
- Notes: NO BLOCK
- Table: NO BLOCK
- Flow-chart: NO BLOCK

GAPS:
- No text extraction mode (must find words in passage)
- No notes format block
- No table completion block
- No flow-chart completion block

RECOMMENDATION: Create specialized blocks:
- TableCompletionBlock.jsx
- FlowChartCompletionBlock.jsx
- NotesCompletionBlock.jsx
Or create a universal CompletionBlock with format prop
