## Issues Found

### Critical
1. **`robots.txt` references wrong domain** ‚Äî Sitemap URL points to `telc.englishexamexercises.com`, but the production domain is `telc.englishexamexercises.com`. The sitemap URL is inconsistently named vs domain branding.
2. **`sitemap.xml` is static and stale** ‚Äî Last modified `2026-04-14`. No dynamic generation. Major public pages under `/free-mock/*`, `/telc/mywords`, `/telc/drillshub`, and all level routes are missing type/priority differentiation.
3. **No `og:image` or `twitter:image`** ‚Äî Social share previews show no image on Facebook, LinkedIn, or X. Shared links display as blank-card or fallback.
4. **SEO component does not handle JSON-LD / structured data** ‚Äî No `application/ld+json` block anywhere for Organization, WebSite, BreadcrumbList, or FAQPage schema. Google rich snippets unavailable.
5. **`BreadcrumbList` schema missing** ‚Äî Hub routes (`/telc/b1/reading`, `/telc/b2/writing`, etc.) have deep paths. Both users and Google benefit from breadcrumb structured data.
6. **`mailto_schema` / no `publisher` / `og:site_name`** ‚Äî Missing `og:site_name` in `index.html` and missing all OG tags from `SEO.jsx`. Also missing `og:image`.
7. **Alt text on dynamically rendered images is generic** ‚Äî `ListeningBlock.jsx` uses `alt="Exam Visual"` for all listening images; `FlowChartCompletionBlock.jsx` and `DiagramLabelBlock.jsx` use `alt={flowchart.alt || 'Flowchart'}` which may be empty.

### High
8. **Meta description duplicated between `index.html` and `SEO.jsx` defaults** ‚Äî `index.html` claims "IELTS, TOEFL, and LangCert" exams, site brand has shifted to "TELC". Inconsistency harms SEO.
9. **`LandingPage.jsx` re-imports robots meta as JSX `<meta>`** ‚Äî Written as `<meta ...>` inside JSX body. JSX does not automatically inject these into `<head>` without `react-helmet` or portal logic. It has no effect.
10. **`<h1>` overuse** ‚Äî LandingPage has one h1 (good), but `BrandTestHub.jsx` uses `<h1>` per card (`<h1 className="hero-title">`, `<h1>` in teacher message), and `ExamDescription.jsx` re-declares h1 per route. Multiple h1s dilute keyword hierarchy.
11. **404 / soft-404 coverage missing** ‚Äî SPA  fallback in `.htaccess` returns `index.html` for all unknown routes. Production should return actual HTTP 404 for truly missing pages or use fallback routing with a `<meta name="robots" content="noindex">`.
12. **No `hreflang` tags** ‚Äî Single-lang site (English) but no explicit `lang` / `hreflang` declarations beyond `lang="en"`. If localised variants are planned, this needs `hreflang` annotations.
13. **No favicon Apple touch / PWA icons** ‚Äî `index.html` points to `/vite.svg`. No `apple-touch-icon`, no `manifest` link beyond PWA auto-injected. iOS shortcuts and Google mobile-first indexing show missing favicon.

### Medium
14. **`keywords` meta tag deprecated by Google** ‚Äî Google ignores it since 2009. Other engines may still use it; can be kept but should not be relied on.
15. **PWA workbox SeoBlocker conflict possible** ‚Äî Workbox `navigateFallbackAllowlist: [/^\/dashboard\//, /^\/test\//]` means non-listed client-side routes (e.g. `/telc/b1/reading`) may be served `index.html` as a SW fallback instead of a proper HTTP response, blocking seo crawler JS execution.
16. **No `noindex` for private/protected routes** ‚Äî Routes behind `ProtectedRoute` (all `/telc/*` app paths) are accessible only to signed-in users. Google can't render them without auth, but they still appear in sitemap with `priority 0.7‚Äì0.8`. Marking them as `noindex` prevents crawl waste.
17. **`index.html` references `english-exam-excellence.com` while site is `telc.englishexamexercises.com`** ‚Äî OG `og:url` and title mention the wrong domain; needs full brand alignment.
18. **`AppShell.jsx` profile avatar has no `alt` attribute variation** ‚Äî `alt="Profile"` on user image. Could be personalized to user name.
19. **No `canonical` on any non-root SPA route** ‚Äî `SEO.jsx` sets it only when explicitly passed. A global `canonical` injection based on `window.location.href` is missing.
20. **`FormComponent`/form elements ‚Äî no accessible labels pass audit** ‚Äî Several `div` buttons with no label from `SEO.jsx` perspective; search input in `BrandTestHub.jsx` line 140 is a bare `<input>` with no associated label (placeholder only).

### Low / Nice-to-Have
21. **Font preload** ‚Äî Google Fonts loaded via `<link>` without `preconnect` / `preload`. First-render FOIT (Flash Of Invisible Text) on slow connections.
22. **No analytics / measurement** ‚Äî No Google Analytics 4 or GTM snippet in `index.html`.
23. **`sitemap.xml` missing `lastmod` auto-generation** ‚Äî Needs build-time or CI/CD script instead of hardcoded dates.

## Recommended Internal Actions (Beyond `SEO.jsx`)

### P1 ‚Äî Meta tag foundation (`index.html`)
- Replace stale domain/brand copy in `index.html` (lines 7‚Äì17) to reflect `telc.englishexamexercises.com` / TELC brand.
- Add `og:image` (1200√ó630px), `twitter:image`, `og:site_name`, `theme-color`, `msapplication-TileColor`.
- Add `favicon` links: `apple-touch-icon`, `icon-192`, `icon-512` (reference PWA icons already in `vite.config.js`).
- Add Google / Bing / Yandex / Naver verification meta tags when keys are available.
- Inject GA4 / GTM snippet.
- Add `preconnect` for Google Fonts origin + `fonts-display=swap`.

### P2 ‚Äî JSON-LD structured data (new file `src/utils/structuredData.jsx`, used in `SEO.jsx`)
- **Organization** schema on every page (organization name, URL, logo, social links).
- **WebSite + SearchAction** on the landing / root page (enables Google sitelinks searchbox).
- **FAQPage** schema on `PricingPage.jsx` (each FAQ Q‚ÜíA pair becomes a schema entry).
- **BreadcrumbList** dynamically generated per route (e.g. `Home ‚Üí TELC B2 ‚Üí Reading`).

### P3 ‚Äî Enhanced `SEO.jsx`
- Auto-add `og:image`, `og:site_name` defaults when not provided.
- Auto-set canonical URL on every route change (derive from `window.location.href`).
- Inject JSON-LD block(s) into `<head>` via a new prop or default payload.
- Clean up duplicate `useEffect` calls ‚Äî merge OG and Twitter updates into one block.
- Add `lang`/`hreflang` management for single and future multi-language routes.

### P4 ‚Äî Accessibility (partial SEO overlap)
- Fix multiple `<h1>` tags: `BrandTestHub.jsx` line 55, 64 should use `<h2>` / `<h3>`.
- Add explicit `<label>` elements to login form controls if present.
- Add descriptive `alt` text to dynamically rendered images (`ListeningBlock.jsx`, `FlowChartCompletionBlock.jsx`, `DiagramLabelBlock.jsx`).
- Give `LandingPage.jsx` JSX `<meta>` tags proper React Helmet integration (using `react-helmet-async`).

### P5 ‚Äî Sitemap + robots.txt
- Regenerate `sitemap.xml` with accurate `lastmod`, covering all public routes.
- Mark protected routes (`/telc/b1/*`, `/telc/b2/*`, `/telc/c1/*`, `/telc/vocabulary/*`, `/telc/drillshub/*`, `/telc/mywords`) as **`<meta name="robots" content="noindex">`** or remove them from sitemap ‚Äî they are behind auth check and will waste crawl budget.
- Sync `robots.txt` pointing to correct sitemap URL (confirm final production domain name). Fix domain mismatch.

### P6 ‚Äî Pre-rendering / SSR consideration
- For a React SPA with `vite-plugin-pwa`, consider adding `vite-plugin-ssr` or a static pre-render script (e.g. a CI step that renders the home page with crawler meta tags) so Googlebot receives fully rendered HTML instead of a blank shell.
- Verify `workbox` `navigateFallbackAllowlist` includes all legitimate SPA scrapes or add explicit `noindex` fallback HTML.

### P7 ‚Äî Performance signals (Core Web Vitals)
- Add `font-display: swap` to Google Fonts `<link href>` tags already in `index.html`.
- Consider self-hosting Plus Jakarta Sans to eliminate third-party font FOIT.
- Enable Brotli on top of gzip (Apache `mod_brotli`) for faster LCP on low-end devices.
- Add `rel="preload"` for above-the-fold images (none exist yet; check when `og:image` is added).
