# Drills Hub Level Integration Plan

## Current State Analysis

### Vocab Hub (Working Model)
- Uses level filtering via `selectedLevel` prop
- Data structure: Categories contain tasks with `level` property (e.g., `{ level: 'B1' }`)
- Component handles two views:
  1. Level selector (when no level selected)
  2. Filtered topic view (when level selected)
- Routes: `/telc/vocabulary`, `/telc/vocabulary/b1`, `/telc/vocabulary/b2`, `/telc/vocabulary/c1`
- Navigation properly handles level context

### Drills Hub (Current State)
- No level filtering implemented
- Data structure: Categories contain tasks without `level` property
- Simple grid layout showing all drills
- Route: `/telc/drillshub` only

## Goal
Connect drills hub to levels like the vocab hub, allowing users to:
- Select a level (B1/B2/C1) to see level-appropriate drills
- Navigate via `/telc/drillshub/{level}` URLs
- Maintain backward compatibility with existing `/telc/drillshub` route

## Implementation Plan

### 1. Data Structure Updates
Add level properties to drill tasks in `src/data/DrillsHub/index.js`:
- Assign appropriate levels to each drill based on difficulty/content
- Ensure each drill has exactly one level (B1, B2, or C1)
- Distribute drills evenly across levels where possible

### 2. Component Updates
Modify `src/components/ui/DrillsHub.jsx` to:
- Accept `selectedLevel` prop (default: null)
- Implement two views like VocabHub:
  a. Level selector grid (when `selectedLevel` is null)
  b. Filtered drill view (when `selectedLevel` is set)
- Filter categories/tasks by `selectedLevel` when applicable
- Add navigation props: `onNavigateToLevelSelector`, `onNavigateToLevel`

### 3. Routing & Navigation Updates
Update `src/App.jsx`:
- Add route handling for `/telc/drillshub/:level`
- Update EXTRA_TOOLS configuration if needed
- Ensure proper navigation between levels and hubs
- Handle `selectedDrillLevel` state similar to `selectedVocabLevel`

### 4. Styling & UX
- Reuse existing level selector styling from VocabHub
- Ensure consistent look and feel
- Maintain existing drill card styling for filtered view

### 5. Backward Compatibility
- Keep existing `/telc/drillshub` route functional (shows all levels)
- Ensure existing drill links still work
- Preserve XP rewards and other metadata
